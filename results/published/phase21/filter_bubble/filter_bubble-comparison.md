## Pre-registration (verbatim, from an arm's `config.json` `description`)

> PHASE 21 SCENARIO filter_bubble (package A). This committed config IS the CONTROL arm; scripts/run_phase21_a.sh patches it into the treatment arm. Both arms share seed 42 and the identical generated world; only exploration.epsilon and the ranking weights differ. HYPOTHESIS: an exploit-only recommender (no exploration, similarity-dominant ranking) progressively narrows users' hidden interest distributions over the 9-day run relative to a recommender running shipped exploration+diversity defaults on the same world -- the classic filter-bubble pathology (V2 TDD 4.18, Tier-4 acceptance re: policy alters hidden preference distributions). MECHANISM: with exploration.epsilon=0 the ExplorationCandidateSource fires no out-of-preference slots, and a similarity-heavy ranking (similarity_weight 0.95 with every novelty/diversity/popularity/creator/exploration ranking signal and the repetition penalty zeroed) serves each user content maximally aligned to their CURRENT hidden preference; satisfaction-driven PreferenceEvolution::applyImpression then reinforces that narrow region every impression, compounding daily. The niche_treasure archetype and small/tail creators -- which reach users mainly via exploration/diversity injection -- lose exposure. EXPECTED SIGNATURE vs control: (1) per-day mean_preference_entropy (trailing column of longterm_metrics.csv) declines faster and to a lower day-8 value; (2) higher mean_preference_shift_from_initial (more distortion); (3) lower whole-run arch_niche_treasure exposure share; (4) lower whole-run tail_creator_share. Early days track loosely; arms diverge as evolution compounds. VERDICT CRITERIA: effect CONFIRMED if day-8 mean_preference_entropy(bubble) < control AND mean_preference_shift_from_initial(bubble) > control AND whole-run arch_niche_treasure(bubble) < control AND whole-run tail_creator_share(bubble) <= control. Report 'effect not observed' or 'partial' honestly if entropy/niche/tail directions fail to hold (e.g. the shipped diversity reranker, still on in both arms, compensates). ARMS: control = THIS config (algorithm hnsw_ranker_diversity COMPLETE mode; exploration.enabled=true, epsilon=0.05; ranking = shipped defaults similarity 0.50 / quality 0.10 / freshness 0.08 / popularity 0.07 / trending 0.08 / creator_affinity 0.07 / exploration 0.05 / repetition_penalty 0.15; diversity on, mmr_lambda 0.75). bubble = identical EXCEPT exploration.epsilon=0.0 and ranking patch {similarity_weight:0.95, quality_weight:0.0, freshness_weight:0.0, popularity_weight:0.0, trending_weight:0.0, creator_affinity_weight:0.0, exploration_weight:0.0, repetition_penalty:0.0}; exploration.enabled STAYS true so the source is registered and its per-slot bernoulli draws stay stream-aligned with control (epsilon=0 fires no slots). Pre-registered before any run per P21-CONTRACTS section 4.

## Notes

- **No aggregate score** (D22 / V2 TDD §6): every group above (engagement, hidden welfare, session health, recommendation quality, long-term, ecosystem) is reported SEPARATELY; this script never combines them into one number, and neither do plot_results.py's Phase 21 additions.

- **Frozen-schema reads only** (contracts §6): every key this script reads is quoted verbatim from contracts §2 (ecosystem) / §3 (longterm_metrics.csv's trailing column) / §5 (long_term, inherited from Phase 20) / the results_writer.cpp-verified V2 §6 group keys -- no candidate-path guessing. A block genuinely absent for an arm (e.g. a control arm run with a different gate set) renders as a clearly labeled n/a cell, never a crash or a silently-guessed alternate key.

- **Per-day mini-table method**: first/middle/final SIMULATED DAY over the UNION of days present in `ecosystem_metrics.csv` and `longterm_metrics.csv` (the two files are gated independently -- contracts §1 requires the event scheduler for the former, §5's P20 gates for the latter -- so an arm with only one of them still gets a partial row).

- **CSV/MD content split**: `<scenario>-comparison.csv` carries only the flat whole-run arms-x-groups table (one row per arm); the per-day mini-table, the verbatim pre-registration quote, and the verdict are `.md`-only (see this script's module docstring).

- **Concurrency-contention caveat**: if arms were run CONCURRENTLY (contracts §4 caps this at 2 simulate processes per package), wall-clock/timing numbers carry cache and memory-bandwidth contention; every other number in this report is deterministic (rng/clock-free, D8/D9) and unaffected by run mode.
